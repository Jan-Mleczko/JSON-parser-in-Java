Dokumentacja parsera JSON Jana Mleczko w Javie

Jan Mleczko, czerwiec-lipiec 2026

Jest to dokumentacja biblioteki klas w języku programowania Java SE 8 (a prawdopodobnie zgodnej także z innymi wersjami Javy) do przetwarzania danych w formacie JSON oraz obliczeń w systemie dziesiętnym. Klasy zamieniają JSON na łatwą w odczycie strukturę obiektową i odwrotnie - umożliwiają tworzenie takiej struktury i zamieniają ją na JSON.

Użytkownik powinien znać format JSON. On sam nie będzie tutaj w pełni opisany.

Zaletą tej biblioteki jest to, że polega ona tylko na standardowych funkcjonalnościach Java SE 8, a nie na innych bibliotekach ani na określonym systemie operacyjnym lub cechach wybranego typu komputera.

Klasy biblioteki zebrane sa w pakiet janmlecz.json.

Wstęp i spis treści Klasa ParserJSON

Metoda parsuj klasy ParserJSON

public static WartoscJSON parsuj (String zakodowany)

Metoda przyjmuje obiekt JSON załadowany w całości do jednego łańcucha znaków - obiektu String. Ma to być pojedyncza wartość dowolnego typu w JSON. Zamieniana jest na obiekt jednej z klas pochodnych od WartoscJSON i ten obiekt jest zwracany przez metodę.

Ten parser nie jest walidatorem. Może przyjąć pewne przypadki nieprawidłowych danych JSON i odwzorować je na strukturę obiektową odpowiadającą jakiemuś prawidłowemu obiektowi JSON.

Rzuca wyjątki:

  • ZleZeSkladniaJSON, gdy tekst nie jest prawidłową wartością w formacie JSON i nie może zostać przeparsowany albo występują w nim zbyt duże lub zbyt dokładne liczby.





Wstęp i spis treści

Klasa WartoscJSON

package janmlecz.json;
public abstract class WartoscJSON

Reprezentuje wartość w sensie JSON dowolnego typu. Dla każdego typu danych JSON jest w bibliotece odpowiednia klasa dziedzicząca po WartoscJSON. Wartości niektórych typów mogą zawierać w sobie zagnieżdżone wartości.

Obiekty WartoscJSON zwracane przez metody biblioteki zawsze są jednej z wyżej wspomnianych klas. Klasa WartoscJSON nie ma żadnych właściwych sobie metod ani pól publicznych. Postępowanie z takim obiektem najlepiej zacząć od ustalenia jego bardziej szczegółowej klasy odpowiadającej typowi wartości pochodzącej z JSON. Można:

  1. rzutować referencję WartoscJSON na referencję jednej z klas pochodnych dla typów JSON. Nieoczekiwany typ objawi się wyjątkiem ClassCastException.
  2. sprawdzić szczegółową klasę operatorem instanceof.

Klasy z biblioteki dziedziczące po WartoscJSON mają to do siebie, że tekst zwracany przez ich metody toString jest poprawnym JSON. Nie dotyczy to obiektów TablicaJSON i SlownikJSON w stanie otwartym, o czym jest mowa w dokumentacji tych klas.

Klasa TablicaJSON

package janmlecz.json;
public class TablicaJSON extends WartoscJSON

Odpowiada tablicy w JSON zapisywanej jako np. [9, 25, "Jan Mleczko"]. Obiekt tej klasy nie jest tworzony ze złożonej struktury danych odpowiadającej tablicy, a zamiast tego wypełniany stopniowo. Obiekt tej klasy jest w jednym z dwóch stanów:

  • otwartym - Jest to stan początkowy pozwalający wypełnić tablicę. Nie jest znany jej rozmiar. Można dopisywać elementy na koniec. Im więcej elementów zostanie dodanych, tym większy będzie rozmiar tablicy. Otwartą tablicę zawsze można domknąć.
  • zamkniętym - Stan końcowy bez powrotu do stanu otwartego. Dostępny jest rozmiar tablicy, w tej chwili już stały, ponieważ nie można już dopisywać elementów.

Jeśli obiekt pochodzi z parsowania JSON, jest już w stanie zamkniętym. Obiekty tworzone konstruktorem do późniejszego umieszczenia w JSON tworzonym za pomocą biblioteki są otwarte, ale do czasu osadzenia w JSON muszą zostać wypełnione wedle potrzeb i domknięte.

Rozmiar tablicy, tzn. ilośc jej elementów, musi mieścić się w zakresie typu int.

Klasa LiczbaDziesietna

package janmlecz.json;
public class LiczbaDziesietna

Obliczenia na dużych liczbach wymiernych w systemie dziesiętnym. Każdy obiekt klasy LiczbaDziesietna reprezentuje pewną liczbę określoną w chwili jego utworzenia i nie zmieniającą się później. Zatem obiekty te są niemutowalne.

Reprezentowane liczby mieszczą się w zakresie od (-1020 + 10-10) do (1020 - 10-10) z krokiem co 10-10.

Zaimplementowane działania to dodawanie, odejmowanie, mnożenie, a także porównywanie. Metody publiczne odpowiedzialne za wykonywanie działań zwracają nowy obiekt reprezentujący wynik (z wyjątkiem oczywiście porównań zwracających wartości logiczne). Jeśli dokładny wynik jest niereprezentowalny, rzucany jest wyjątek.

Metoda naTekst klasy LiczbaDziesietna

public String naTekst (int wyrownanieCalkowitej, int wyrownanieUlamkowej,
  char znakPodzialu, char znakWyrownania, boolean zaznaczacPlus)

Zamienia liczbę dziesiętną na tekst z możliwościami formatowania.

wyrownanieCalkowitej - szerokość pola w znakach, do jakiej ma być wyrównana część całkowita, jeśli jest krótsza. Zero dla braku wyrównania. Przy zerowej wartości całkowitej, przyjmowane jest, że ma ona zero cyfr.

wyrownanieUlamkowej - szerokość pola w znakach, do jakiej ma być wyrównana część ułamkowa, jesli jest krótsza. Zero dla braku wyrównania.

znakPodzialu - znak oddzielający część ułamkową od całkowitej, np. kropka lub przecinek.

znakWyrownania - znak, którym dopełniana jest długość zapisu przy wyrównywaniu, np. spacja lub cyfra zero.

zaznaczacPlus - czy liczby nieujemne mają być poprzedzane znakiem "+". Znak liczby nie liczy się do szerokości przy wyrównywaniu.

Metoda toString klasy LiczbaDziesietna

public String toString ()

Przedstawia liczbę dziesiętną w krótkiej formie tekstowej będącą prawidłową wartością JSON oraz jednocześnie nadającą się do zamiany na wartość zmiennoprzecinkową przez metodę standardową Float.parseFloat, o ile wartość jest reprezentowalna przez typ float.

Wartością zwracaną tej metody może być np. 12345000000 albo -0.5.
